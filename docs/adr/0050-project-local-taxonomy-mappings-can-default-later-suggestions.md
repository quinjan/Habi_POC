# Project-Local Taxonomy Decisions Can Default Later Suggestions

Approved and mapped Taxonomy Decisions may influence later default categorization only inside the same Project Workspace. Approved decisions can default later matching suggestions to the approved project taxonomy node, while mapped decisions use the original AI-suggested path as the remembered project-local mapping. For a normalized suggested path, the latest retained Taxonomy Decision wins: a later approved or mapped decision can replace earlier rejection context for future defaults. When Habi preselects a category because of either kind of prior Taxonomy Decision, the review UI should disclose that provenance with hover or info text so the reviewer can distinguish reused project knowledge from a fresh AI suggestion.
