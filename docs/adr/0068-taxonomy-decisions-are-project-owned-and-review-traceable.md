# Taxonomy Decisions Are Project Owned And Review Traceable

Each Taxonomy Decision belongs to exactly one Project Workspace, and decisions made while resolving a Taxonomy Gate should reference their originating Review Batch and affected AI-suggested category path. A Taxonomy Decision may only be created for a normalized AI-suggested path that actually appears among candidates in that referenced Review Batch and Project Workspace, so the endpoint cannot become a generic taxonomy-learning backdoor. The decision's learning effect remains project-scoped, while the review-batch origin preserves traceability for reviewers inspecting batch history. Approved taxonomy learning survives a later review closed with no import status because the reviewer explicitly approved project taxonomy growth independent of importing candidate memory records.
