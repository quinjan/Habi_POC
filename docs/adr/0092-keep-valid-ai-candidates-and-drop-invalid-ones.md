# Keep Valid AI Candidates And Drop Invalid Ones

When AI Extraction returns a mix of valid and invalid Purchase Line candidate proposals, Habi will keep the valid candidates and drop the invalid ones before creating the Review Batch. A candidate is invalid when it cannot safely enter review, such as missing a useful name, using a line type other than `material` or `service`, having malformed currency or date fields, lacking candidate-level confidence, failing to reference the Source Submission evidence, or including out-of-scope direct memory writes, duplicate suggestions, or non-Purchase-Line candidate types. If no returned candidates are valid, the Processing Job should become `failed` rather than `no_candidates_found` because the AI pipeline produced unusable structured output, not a trustworthy empty result.
