# Resolved Category Paths Are Two Levels For POC

For the POC, a Resolved Category Path consists of exactly one Top-Level Category and one subcategory. Approved Taxonomy Decisions must include both path segments, and mapped Taxonomy Decisions must point to an existing subcategory leaf node in the same Project Workspace rather than a Top-Level Category alone. Habi will not support arbitrary-depth taxonomy paths in this slice because deeper hierarchies would complicate Taxonomy Gates, mapping, uniqueness rules, review UI, and import display beyond the current product need.
